== What you demonstrated

You proved that **intelligent cache-aware routing reduces tail latency** for your most frustrated users—using the same hardware and the same workload throughout:

* **Single GPU baseline**: Established P95/P99 TTFT under concurrent load using a KV-cache-aware prompts file
* **Naive 4 GPU scaling**: 4x more throughput capacity, but P95 tail latency improved only marginally due to cache misses from random routing
* **llm-d with 4 GPUs**: `prefix-cache-scorer` routing drove cache hit rates to 90%+, dramatically reducing P95/P99 TTFT

The key insight: **more GPUs improve capacity, but intelligent cache-aware routing improves customer experience**.

== When to use llm-d

Prefix-aware routing with llm-d provides the most value when your workload has:

* Repeated prompt prefixes (system prompts, shared documents, common contexts)
* High concurrency where cache reuse matters at scale
* Tail latency requirements that matter for customer experience
